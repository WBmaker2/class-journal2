# [Design] 모바일 반응형 디자인 최적화 (v3.6)

## 1. 반응형 전략
- **Mobile First**: 기본 스타일을 모바일(작은 화면)에 맞추고, `md`, `lg` 브레이크포인트를 사용해 데스크톱 화면을 확장하는 방식 적용.
- **Breakpoints**: 
    - `sm`: 640px (핸드폰 가로)
    - `md`: 768px (태블릿)
    - `lg`: 1024px (데스크톱)

## 2. 컴포넌트별 상세 설계

### 2.1 메인 헤더 및 네비게이션 (`App.tsx`)
- **모바일**: 로고와 동기화 인디케이터를 세로로 배치하거나, 공간이 부족할 경우 로고 크기 축소.
- **클래스 선택기**: `flex-wrap`을 적용하여 학급 버튼이 여러 줄로 자연스럽게 내려오도록 수정.

### 2.2 대시보드 그리드 (`Dashboard.tsx`)
- **구조**:
    - Mobile: `grid-cols-1` (카드들이 한 줄로 나열)
    - Tablet: `grid-cols-2`
    - Desktop: `grid-cols-3` 또는 사이드바 포함 구조.
- **패딩**: 모바일에서는 카드 사이의 간격과 내부 패딩을 줄여 공간 효율성 극대화.

### 2.3 시간표 (`TimetableManager.tsx`)
- **문제**: 가로로 긴 5일치 시간표는 모바일에서 잘림.
- **해결**: 
    - 부모 컨테이너에 `overflow-x-auto` 적용하여 수평 스크롤 가능하게 처리.
    - 폰트 크기(`text-xs`) 및 셀 너비 조정.
    - 현재 요일 강조 및 가시성 확보.

### 2.4 학급 일지 에디터 (`ClassLogEditor.tsx`)
- **모바일**: 텍스트 입력창이 화면을 꽉 채우도록 `w-full` 적용.
- **버튼**: '저장', '초기화' 버튼을 모바일에서는 하단에 고정하거나 더 크게 배치하여 터치 편의성 제공.

### 2.5 모달 및 UI 컴포넌트 (`src/components/ui/`)
- **Modal**: 모바일에서는 `w-[95%]` 또는 거의 전체 화면을 차지하도록 너비 조정.
- **Button**: 터치 타겟 확보를 위해 모바일에서 최소 높이 40~44px 보장.

## 3. 스타일 가이드 (Tailwind)
- **Container**: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8` 적용하여 양옆 여백 관리.
- **Typography**: 모바일에서는 기본 폰트 크기를 약간 줄이고(`text-sm`), 줄 간격을 조절하여 가독성 유지.

## 4. 구현 우선순위
1.  `App.tsx` 최상단 레이아웃 반응형 대응.
2.  `Dashboard` 카드 그리드 최적화.
3.  `Timetable` 수평 스크롤 및 크기 조정.
4.  기타 관리 모달(학급/학생 관리) 너비 조정.
