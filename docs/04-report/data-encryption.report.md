# [Report] 종단간 암호화 및 데이터 무결성 강화 (v3.7.6)

## 1. 프로젝트 개요
- **목표**: 사용자 데이터를 AES-256 알고리즘으로 암호화하여 개인정보 보호를 강화하고, 체크섬을 통한 데이터 무결성 검증 및 보안 비밀번호 정책 강화를 수행함.
- **결과**: 종단간 암호화(E2EE), SHA-256 기반 체크섬 검증, 강화된 비밀번호 정책 구현 완료.

## 2. 구현 내용
### 2.1 암호화 알고리즘 및 키 관리
- **AES-256 (CryptoJS)**: 모든 로컬 데이터를 클라우드 업로드 전 AES-256 방식으로 암호화.
- **클라이언트 사이드 키 관리**: 보안 비밀번호는 서버로 전송되지 않으며 `sessionStorage`에만 임시 보관하여 메모리 상의 보안 유지.

### 2.2 데이터 무결성 검증 (Checksum)
- **SHA-256 Checksum**: 데이터 업로드 시 원본 데이터의 SHA-256 해시값을 생성하여 함께 저장.
- **복호화 후 검증**: 데이터 다운로드 및 병합 시, 복호화된 데이터의 해시값을 재계산하여 서버에 저장된 체크섬과 비교함으로 데이터 변조나 손상 여부를 완벽히 차단.

### 2.3 보안 비밀번호 정책 강화
- **복잡도 요구사항**: 최소 8자리 이상, 영문과 숫자의 혼합 사용 강제.
- **UI/UX 반영**: `SecurityKeyModal` 내 실시간 검증 알림 및 명확한 가이드 제공.

## 4. 향후 과제
- **PBKDF2 키 파생**: 사용자 비밀번호를 그대로 키로 사용하는 대신 salt와 반복 횟수를 포함한 PBKDF2를 적용하여 무차별 대입 공격에 대한 저항력 강화.
- **암호화 전환 가이드**: 암호화되지 않은 기존 데이터를 보유한 사용자를 위한 암호화 마이그레이션 UI 추가 고려.
