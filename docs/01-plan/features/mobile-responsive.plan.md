# [Plan] 모바일 반응형 디자인 최적화 (v3.6)

## 1. 개요
- **기능명**: 모바일 반응형 UI/UX 최적화
- **목표**: 핸드폰, 태블릿 등 다양한 해상도에서 모든 기능(학급 일지, 시간표, 관리 도구)을 쾌적하게 사용할 수 있도록 UI 개선.
- **주요 개선 대상**:
    1. **메인 레이아웃**: 상단 네비게이션 및 대시보드 카드 배열 조정.
    2. **시간표**: 모바일에서 가로 스크롤 또는 세로형 뷰 제공.
    3. **입력 폼**: 모바일 가상 키보드 대응 및 버튼 크기 최적화 (Touch-friendly).
    4. **모달/다이얼로그**: 작은 화면에서 꽉 차는 풀스크린 모달 적용.

## 2. 사용자 스토리
- "선생님이 이동 중이나 교실 뒤편에서 핸드폰으로 급하게 일지를 기록할 때, 화면이 잘리거나 버튼이 너무 작지 않았으면 좋겠어."
- "태블릿에서도 대시보드가 한눈에 들어오고, 터치 조작이 쉬웠으면 좋겠어."

## 3. 상세 요구사항
### 3.1 레이아웃 및 네비게이션
- 상단 헤더의 로고와 인디케이터 배치를 `flex-col` 또는 `flex-wrap`으로 대응.
- 모바일용 하단 탭 바 또는 햄버거 메뉴 검토 (현재 구조 유지하며 좁은 공간 최적화 우선).

### 3.2 시간표 및 일지 에디터
- **시간표**: 모바일 환경에서 폰트 크기 축소 및 수평 스크롤 강제 적용.
- **일지 에디터**: 텍스트 영역(textarea) 높이 자동 조절 및 모바일 전체 너비 사용.

### 3.3 대시보드 위젯
- `grid-cols-1` (모바일) -> `grid-cols-2` (태블릿) -> `grid-cols-3` (데스크톱) 반응형 그리드 강화.
- 카드 내부 패딩 및 마진 최적화.

## 4. 제약 사항 및 고려 사항
- **기존 기능 유지**: 데스크톱 사용자의 편리함을 해치지 않는 선에서 점진적 개선(Progressive Enhancement).
- **터치 타겟**: 버튼 높이를 최소 44px 이상으로 확보하여 오클릭 방지.

## 5. 성공 기준
- [ ] iPhone SE 등 작은 화면에서도 좌우 잘림 없이 모든 콘텐츠가 표시됨.
- [ ] 모바일에서 버튼 클릭이 쉽고 텍스트 읽기가 편해짐.
- [ ] 가로/세로 모드 전환 시 레이아웃이 유연하게 대응함.
