# [Plan] 실시간 동기화 기능 개선 (Real-time Synchronization Improvement)

## 1. 개요 (Overview)
여러 기기에서 동일한 계정으로 로그인하여 사용할 때, 데이터 유실 없이 실시간으로 동기화가 이루어지도록 기능을 개선합니다. 특히 기기 B가 로그인하거나 포커스를 가질 때, 클라우드의 데이터가 로컬보다 최신인 경우 자동으로 다운로드하거나 충돌 해결 절차를 거치도록 합니다.

## 2. 목표 (Goals)
- [ ] 로그인 시 클라우드 데이터의 최종 수정 시간 확인 및 자동 동기화
- [ ] 앱 포커스 시 실시간 데이터 변경 감지 및 동기화 프로세스 강화
- [ ] 로컬 변경 사항이 없을 때(`isDirty`=false) 최신 원격 데이터 자동 적용
- [ ] 로컬/원격 양쪽 모두 변경 사항이 있을 때 충돌 해결 모달 표시 및 병합 지원

## 3. 요구사항 분석 (Requirements Analysis)
- **최신성 확인:** `user_journal_data` 테이블의 `updated_at` 필드를 사용하여 로컬의 `lastSyncTime`과 비교합니다.
- **자동 다운로드 조건:** 로컬에 아직 반영되지 않은 최신 데이터가 있고, 로컬에서 수정 중인 내용(`isDirty`)이 없는 경우 사용자 개입 없이 자동 다운로드합니다.
- **충돌 해결:** 로컬에서 수정 중인 내용이 있는데 원격에 더 최신 데이터가 발견된 경우, '병합', '덮어쓰기', '강제 업로드' 중 선택할 수 있는 UI를 제공합니다.
- **실시간성:** Supabase Realtime을 통해 데이터 변경 이벤트를 구독하고 즉시 동기화 로직을 트리거합니다.

## 4. 기술적 설계 (Technical Design)
- **대상 파일:**
  - `src/context/SupabaseContext.tsx`: 주요 동기화 및 충돌 처리 로직
  - `src/services/supabase.ts`: `fetchUserData` 등의 API 호출 로직
- **주요 로직 변경:**
  - `checkRemoteVersion` 함수를 개선하여 단순 알림이 아닌 상태에 따른 자동 동기화(Download) 수행.
  - `downloadData` 함수에 무결성 검증 및 로컬 상태 업데이트 강화.
  - 로그인 성공 직후 `downloadData`를 호출하기 전, 원격 데이터의 존재 여부와 수정 시간을 먼저 확인하도록 최적화.

## 5. 작업 계획 (Tasks)
1. [ ] `SupabaseContext` 내 `checkRemoteVersion` 로직 강화 (자동 다운로드 로직 추가)
2. [ ] 로그인 직후 동기화 시퀀스 최적화
3. [ ] 로컬 데이터 수정 시(`markAsDirty`) 실시간 동기화와의 간섭 최소화 테스트
4. [ ] 다중 기기 환경에서의 시나리오 테스트 (A 수정 -> B 확인)

## 6. 기대 결과 (Expected Outcomes)
- 여러 기기에서 작업 시 데이터 덮어쓰기로 인한 손실 방지
- 수동 동기화 버튼 클릭 없이도 최신 데이터 유지
- 사용자 경험 향상 및 데이터 신뢰성 확보
