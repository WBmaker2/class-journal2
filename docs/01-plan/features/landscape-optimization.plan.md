# [Plan] 가로 모드 및 대화면 최적화 (v3.8)

## 1. 개요
- **기능명**: 가로 모드 및 대화면(태블릿/폴더블) 최적화
- **목표**: 태블릿 및 폴더블 기기의 가로 모드에서 넓은 화면 공간을 효율적으로 활용하여 더 많은 정보를 한눈에 보여줄 수 있도록 전용 레이아웃 고도화.
- **주요 개선 대상**:
    1. **사이드바 레이아웃**: 대화면 가로 모드에서 사이드바 상시 노출(Split-view) 및 네비게이션 편의성 강화.
    2. **대시보드 위젯**: 넓은 공간을 활용한 3열 이상의 그리드 배치 및 상세 정보 노출.
    3. **2분할 뷰(Dual-pane)**: 일지 에디터와 출결 현황 또는 시간표를 한 화면에 동시에 띄우는 레이아웃 검토.
    4. **시간표 확장**: 가로 모드에서 스크롤 없이 주간 시간표 전체를 고해상도로 표시.

## 2. 사용자 스토리
- "태블릿을 가로로 놓고 쓸 때는 화면 양옆 공간이 많이 남는데, 이 공간에 다른 메뉴나 정보를 같이 볼 수 있으면 좋겠어."
- "일지를 쓰면서 동시에 출석부나 시간표를 옆에 띄워놓고 참고하고 싶어."

## 3. 상세 요구사항
### 3.1 레이아웃 엔진 고도화
- `xl` (1280px 이상) 및 `2xl` (1536px 이상) 브레이크포인트 적극 활용.
- 가로 모드 감지(`orientation: landscape`)에 따른 특화 스타일 적용.

### 3.2 2분할/다중 열 레이아웃
- 대화면 가로 모드에서 사이드바를 숨기지 않고 왼쪽에 고정하여 퀵 네비게이션 제공.
- 에디터 화면에서 우측 여백을 활용하여 '참조 위젯'(오늘의 출결, 시간표 등) 배치.

### 3.3 콘텐츠 밀도 최적화
- 넓은 화면에서 폰트 크기 및 요소 간 간격을 조정하여 정보 밀도 상향.
- 대화면 전용 데이터 시각화 차트 확장.

## 4. 제약 사항 및 고려 사항
- **반응형 연속성**: 세로 모드로 전환하거나 창 크기를 줄였을 때 기존 모바일 최적화 레이아웃으로 부드럽게 전환되어야 함.
- **성능**: 한 화면에 많은 컴포넌트가 노출되므로 렌더링 성능 최적화 필요.

## 5. 성공 기준
- [ ] 태블릿 가로 모드에서 사이드바와 메인 콘텐츠가 조화롭게 배치됨.
- [ ] 넓은 화면 공간이 낭비되지 않고 유의미한 정보로 채워짐.
- [ ] 2분할 레이아웃을 통해 작업 효율성(일지 작성 중 정보 참조 등)이 향상됨.
